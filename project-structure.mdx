## app/

The `app/` directory is the Next.js App Router root. Every subdirectory with a `page.tsx` becomes a URL route.

### `app/layout.tsx`

The root layout wraps every page in the application. It:

- Loads the **Geist Sans** and **Geist Mono** Google Fonts and exposes them as CSS variables (`--font-geist-sans`, `--font-geist-mono`).
- Injects an inline `<script>` that runs **before first paint** to read `localStorage` and apply `data-theme` and `data-color` attributes on `<html>`, preventing a theme flash on page load.
- Sets `suppressHydrationWarning` on both `<html>` and `<body>` to silence the expected hydration mismatch caused by the theme script.

```tsx
const themeScript = `
  try {
    const root = document.documentElement;
    const savedTheme = localStorage.getItem('theme');
    if (savedTheme) {
      root.setAttribute('data-theme', savedTheme);
    } else {
      root.setAttribute('data-theme', 'dark');
    }
    const color = localStorage.getItem('color') || 'violet';
    root.setAttribute('data-color', color);
  } catch (e) {}
`;
```

### `app/globals.css`

The central design system file. It defines all CSS custom properties organised by semantic section (`--header-*`, `--hero-*`, `--feat-*`, `--price-*`, `--footer-*`) and maps them to Tailwind's `@theme` layer. Each `data-theme` and `data-color` attribute combination overrides the relevant token set.

### `app/page.tsx`

The home page. It is a `"use client"` component that composes all landing page sections in order, wrapping each in a `FadeInSection` helper that applies a Motion `whileInView` entrance animation:

```tsx
function FadeInSection({ children }: { children: React.ReactNode }) {
  return (
    <motion.div
      initial={{ opacity: 0, y: 16 }}
      whileInView={{ opacity: 1, y: 0 }}
      viewport={{ once: true, margin: "-80px" }}
      transition={{ duration: 0.5, ease: [0.25, 0.1, 0.25, 1] }}
      style={{ willChange: "opacity, transform" }}
    >
      {children}
    </motion.div>
  );
}
```

## Key configuration files

### `next.config.ts`

Configures Next.js image optimisation to allow remote images from two external hostnames:

```ts
const nextConfig: NextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: "https",
        hostname: "pub-940ccf6255b54fa799a9b01050e6c227.r2.dev",
      },
      {
        protocol: "https",
        hostname: "assets.aceternity.com",
      },
    ],
  },
};
```

Add any additional `remotePatterns` entries here when using images hosted on external domains.

### `tsconfig.json`

TypeScript is configured for strict mode with `"module": "esnext"` and `"moduleResolution": "bundler"` to match Next.js 16's expectations. The `@/*` path alias maps to the repository root, enabling imports like `@/components/hero` throughout the project.

```json
{
  "compilerOptions": {
    "strict": true,
    "module": "esnext",
    "moduleResolution": "bundler",
    "paths": {
      "@/*": ["./*"]
    }
  }
}
```

### `postcss.config.mjs`

Registers the official Tailwind CSS v4 PostCSS plugin. No additional PostCSS plugins are required.

```js
const config = {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

### `eslint.config.mjs`

Uses ESLint's new flat config format with the Next.js Core Web Vitals and TypeScript rule sets. The `.next/`, `out/`, and `build/` directories are ignored.

```js
import { defineConfig, globalIgnores } from "eslint/config";
import nextVitals from "eslint-config-next/core-web-vitals";
import nextTs from "eslint-config-next/typescript";

const eslintConfig = defineConfig([
  ...nextVitals,
  ...nextTs,
  globalIgnores([".next/**", "out/**", "build/**", "next-env.d.ts"]),
]);
```

### `components.json`

Tracks the shadcn/ui configuration used when scaffolding new UI primitives. It points to `app/globals.css` for CSS variables, uses the `new-york` style variant, and registers several third-party component registries (Aceternity, Magic UI, React Bits, and others) for use with the shadcn CLI.
