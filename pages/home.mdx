The home page assembles nine components in a vertical stack, each wrapped in a `FadeInSection` scroll-triggered animation. It is defined in `app/page.tsx` and marked `"use client"`.

## FadeInSection wrapper

Every section in `<main>` is wrapped in a local `FadeInSection` component defined at the top of `app/page.tsx`:

```tsx
function FadeInSection({ children }: { children: React.ReactNode }) {
  return (
    <motion.div
      initial={{ opacity: 0, y: 16 }}
      whileInView={{ opacity: 1, y: 0 }}
      viewport={{ once: true, margin: "-80px" }}
      transition={{ duration: 0.5, ease: [0.25, 0.1, 0.25, 1] }}
      style={{ willChange: "opacity, transform" }}
    >
      {children}
    </motion.div>
  );
}
```

- `initial` — sections start fully transparent and shifted 16 px down.
- `whileInView` — they animate to full opacity and their natural position once they enter the viewport.
- `viewport={{ once: true, margin: "-80px" }}` — the animation fires once, triggered 80 px before the element reaches the visible area.
- `transition` — 500 ms with a cubic bezier ease-in-out curve.

<Note>
  `FadeInSection` is not exported. If you want to reuse this animation elsewhere, move it to a shared file under `components/`.
</Note>

## Customization

**Reorder sections** — Cut and paste the `<FadeInSection>` blocks inside `<main>` in `app/page.tsx`. The render order determines visual order.

**Remove a section** — Delete the corresponding `<FadeInSection>` block. You do not need to modify any other file.

**Add a section** — Import your component at the top of `app/page.tsx`, then wrap it in `<FadeInSection>` and place it at the desired position inside `<main>`.

```tsx
import MyNewSection from "@/components/my-new-section";

// Inside <main>:
<FadeInSection>
  <MyNewSection />
</FadeInSection>
```

**Disable the animation** — Replace `<FadeInSection>` with a plain `<div>` wrapper if you prefer no scroll animation on a particular section.
