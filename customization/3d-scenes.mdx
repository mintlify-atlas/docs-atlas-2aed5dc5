## How Spline scenes work

Arcoms 1 embeds interactive 3D scenes using [@splinetool/react-spline](https://spline.design). The scene is loaded with Next.js dynamic import and `ssr: false` to prevent server-side rendering — Spline's WebGL renderer requires a browser context and cannot run on the server.

```tsx components/hero.tsx
import dynamic from "next/dynamic";

const SplineScene = dynamic(
  () => import("@/components/ui/splite").then((mod) => mod.SplineScene),
  { ssr: false },
);
```

The scene URL used in the hero is:

```
https://prod.spline.design/kZDDjO5HuC9GJUM2/scene.splinecode
```

It is passed directly to the `SplineScene` component via the `scene` prop:

```tsx components/hero.tsx
<SplineScene
  scene="https://prod.spline.design/kZDDjO5HuC9GJUM2/scene.splinecode"
  className="w-full h-full"
/>
```

## SplineScene component

The `SplineScene` wrapper in `components/ui/splite.tsx` uses React's `lazy` and `Suspense` to code-split the Spline library and show a loader until the scene is ready:

```tsx components/ui/splite.tsx
"use client";

import { Suspense, lazy } from "react";

const Spline = lazy(() => import("@splinetool/react-spline"));

interface SplineSceneProps {
  scene: string;
  className?: string;
}

export function SplineScene({ scene, className }: SplineSceneProps) {
  return (
    <Suspense
      fallback={
        <div className="w-full h-full flex items-center justify-center">
          <span className="loader"></span>
        </div>
      }
    >
      <Spline scene={scene} className={className} />
    </Suspense>
  );
}
```

The `fallback` renders a `.loader` spinner while the Spline bundle and scene file are being fetched. Once both are ready, the scene replaces the spinner.

## Replacing a scene

<Steps>
  <Step title="Create your scene in Spline">
    Open [spline.design](https://spline.design) and build or import your 3D scene. Spline's editor supports objects, materials, lighting, animations, and pointer events.
  </Step>
  <Step title="Export the scene URL">
    In the Spline editor, click **Export** → **Spline Viewer** → **Copy URL**. The URL will have the format:

    ```
    https://prod.spline.design/<scene-id>/scene.splinecode
    ```

    You can also use a self-hosted `.splinecode` file if you download the scene export and serve it as a static asset.
  </Step>
  <Step title="Update hero.tsx">
    Replace the `scene` prop value in `components/hero.tsx` with your new URL:

    ```tsx components/hero.tsx
    <SplineScene
      scene="https://prod.spline.design/YOUR_SCENE_ID/scene.splinecode"
      className="w-full h-full"
    />
    ```

    The surrounding transform classes (`scale-[1.04]`, `-translate-y-16`, `translate-x-[285px]`) position and scale the scene within the hero frame. Adjust these to fit your scene's composition:

    ```tsx components/hero.tsx
    <div className="w-full h-full scale-[1.04] origin-top -translate-y-16 translate-x-[285px]">
      <SplineScene
        scene="https://prod.spline.design/YOUR_SCENE_ID/scene.splinecode"
        className="w-full h-full"
      />
    </div>
    ```
  </Step>
</Steps>

<Tip>
Keep Spline scenes lightweight. Large scenes with many polygons or high-resolution textures significantly increase initial load time. Aim for scene files under 5 MB.
</Tip>

## Performance notes

**`ssr: false`** — The dynamic import with `ssr: false` ensures the Spline bundle (which is large) is never included in the server-rendered HTML or the initial JS bundle sent to the client. It only loads after hydration.

**Lazy loading** — `React.lazy` inside `SplineScene` further splits the Spline library into a separate chunk, which is only fetched when the component first renders in the browser.

**Initial opacity-0 until loaded** — The hero wraps the scene in a container that starts invisible and transitions to visible after a short delay, ensuring the layout does not flash or shift while the scene loads:

```tsx components/hero.tsx
const [isVisible, setIsVisible] = useState(false);

useEffect(() => {
  const t = setTimeout(() => setIsVisible(true), 80);
  return () => clearTimeout(t);
}, []);

// Applied to the scene container:
className={`
  absolute inset-0 z-10
  transition-[opacity,transform] duration-[1600ms] ease-out will-change-[opacity,transform]
  ${isVisible ? "opacity-100 translate-y-0" : "opacity-0 translate-y-10"}
`}
```

The 80 ms delay and 1600 ms fade give the scene time to initialize before it becomes visible, preventing a WebGL flicker on first render.

## Cursor glow effect

The hero card tracks the mouse position within its bounds and exposes it as CSS custom properties `--x` and `--y`. These can be used in CSS to create a radial glow that follows the cursor.

```tsx components/hero.tsx
const cardRef = useRef<HTMLDivElement>(null);

useEffect(() => {
  const el = cardRef.current;
  if (!el) return;

  let rafId: number;
  const move = (e: MouseEvent) => {
    cancelAnimationFrame(rafId);
    rafId = requestAnimationFrame(() => {
      if (!el) return;
      const rect = el.getBoundingClientRect();
      const x = e.clientX - rect.left;
      const y = e.clientY - rect.top;

      el.style.setProperty("--x", `${x}px`);
      el.style.setProperty("--y", `${y}px`);
    });
  };

  el.addEventListener("mousemove", move, { passive: true });
  return () => {
    el.removeEventListener("mousemove", move);
    cancelAnimationFrame(rafId);
  };
}, []);
```

The handler is wrapped in `requestAnimationFrame` so position updates are batched to the browser's render cycle, preventing excessive style recalculations during fast mouse movement. The event listener uses `{ passive: true }` to allow the browser to handle scroll events concurrently without waiting for the handler.

To use the values in CSS, reference `--x` and `--y` with a `radial-gradient`:

```css
.hero-card {
  background: radial-gradient(
    600px circle at var(--x) var(--y),
    rgba(255, 255, 255, 0.04),
    transparent 40%
  );
}
```
