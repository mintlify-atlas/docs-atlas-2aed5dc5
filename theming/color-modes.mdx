---
title: "Color modes"
description: "Light, Dark, and Colorblind modes — what each one changes and the CSS tokens that define them."
---

Arcoms 1 ships with three color modes selectable via the `data-theme` attribute on the `<html>` element. Each mode defines a complete set of CSS custom properties that components consume through semantic tokens.

The `ThemeSwitcher` button in the header cycles through the three modes in order: `light → dark → colorblind`. You can also set a mode directly — see [Switching modes](#switching-modes) below.

## Light mode

Light mode is the default for first-time visitors if no `localStorage` preference exists (though the initialization script defaults to `dark` — see [Theming overview](/theming/overview)).

Applied by `:root` and `[data-theme="light"]`. The palette uses a white background with near-black text.

```css globals.css
[data-theme="light"] {
  --background: #ffffff;
  --foreground: #0f0f0f;
  --card: #f8f8f8;
  --card-foreground: #0f0f0f;

  --primary: #000000;
  --primary2: #ffffff;
  --primary-general: #000000;       /* section headings */
  --primary-color-text: #4b4b4b;    /* descriptive body text */
  --primary-header: #363636;        /* header hover state */

  --muted: #f5f5f4;
  --muted-foreground: #52525b;
  --secondary: #f5f5f4;
  --border: #e7e5e4;
  --input: #e7e5e4;

  --img-frame: rgba(0, 0, 0, 0.1);
}
```

<Note>
The `--primary-color-text` value (`#4b4b4b`) is tuned for readable contrast against the `#ffffff` background on description text throughout sections like Hero, Features, and Pricing.
</Note>

## Dark mode

Dark mode is the out-of-the-box default. The background is pure black (`#000000`) with near-white text.

Applied by `[data-theme="dark"]`.

```css globals.css
[data-theme="dark"] {
  --background: #000000;
  --foreground: #fafafa;
  --card: #090909;
  --card-foreground: #fafafa;

  --primary: #4e4e4e;
  --primary2: #000000;
  --primary-general: #ffffffeb;     /* section headings */
  --primary-color-text: #bababa;    /* descriptive body text */
  --primary-header: #ffffff;        /* header hover state */

  --muted: #1c1917;
  --muted-foreground: #a8a29e;
  --secondary: #1c1917;
  --border: #1c1917;
  --input: #1c1917;

  --img-frame: rgba(255, 255, 255, 0.12);
}
```

<Tip>
When an accent color is active in dark mode, the `--primary` token is replaced by the accent value (e.g., `#4600e9` for violet), while the rest of the dark palette stays intact.
</Tip>

## Colorblind mode

Colorblind mode uses a high-contrast blue-and-yellow palette designed to remain distinguishable under common forms of color vision deficiency. All accent-like colors are replaced by `#2563eb` (Tailwind blue-600), and `--secondary` / `--accent` use `#facc15` (Tailwind yellow-400) for contrast.

Applied by `[data-theme="colorblind"]`.

```css globals.css
[data-theme="colorblind"] {
  --background: #ffffff;
  --foreground: #111827;
  --card: #ffffff;
  --card-foreground: #111827;

  --primary: #2563eb;
  --primary2: #ffffff;
  --primary3: #2563eb;
  --primary4: #2563eb;
  --primary-general: #2563eb;       /* section headings use blue */
  --primary-color-text: #374151;    /* descriptive body text */
  --primary-header: #2563eb;        /* header hover state */

  --secondary: #facc15;             /* yellow accent surface */
  --secondary-foreground: #111827;
  --accent: #facc15;                /* yellow accent */
  --accent-foreground: #111827;

  --muted: #e5e7eb;
  --muted-foreground: #4b5563;
  --border: #d1d5db;
  --input: #d1d5db;

  /* All three card-color slots unified to blue */
  --primary-color-1: #2563eb;
  --primary-color-2: #2563eb;
  --primary-color-3: #2563eb;
}
```

<Info>
Colorblind mode defines `--primary` and related tokens as `#2563eb`. However, accent color selectors (`[data-color="orange"]` etc.) come after `[data-theme="colorblind"]` in `globals.css` and have equal CSS specificity, so they will override the colorblind primary tokens. The high-contrast background and surface tokens (white background, gray borders) remain unchanged by accent overrides.
</Info>

## Switching modes

Use `setAttribute` to change the mode at runtime:

<CodeGroup>

```js Switch to light
document.documentElement.setAttribute('data-theme', 'light');
localStorage.setItem('theme', 'light');
```

```js Switch to dark
document.documentElement.setAttribute('data-theme', 'dark');
localStorage.setItem('theme', 'dark');
```

```js Switch to colorblind
document.documentElement.setAttribute('data-theme', 'colorblind');
localStorage.setItem('theme', 'colorblind');
```

</CodeGroup>

To cycle through modes programmatically (matching the `ThemeSwitcher` behavior):

```js
const modes = ['light', 'dark', 'colorblind'];
const current = document.documentElement.getAttribute('data-theme') || 'dark';
const next = modes[(modes.indexOf(current) + 1) % modes.length];
document.documentElement.setAttribute('data-theme', next);
localStorage.setItem('theme', next);
```
