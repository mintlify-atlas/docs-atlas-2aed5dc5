title: "CSS tokens reference"
description: "Complete reference of all CSS custom properties in globals.css, organized by component section."
---

All design tokens are CSS custom properties defined in `globals.css`. They follow a two-layer architecture:

1. **Primitive tokens** — raw values like `--background`, `--foreground`, `--primary`. Set per color mode and overridden by accent color selectors.
2. **Semantic component tokens** — component-scoped aliases like `--header-bg`, `--feat-card`, `--price-btn-txt`. These reference primitives and are what components consume directly.

Components reference semantic tokens via Tailwind's `@theme inline` registration, which maps every `--token` to a `--color-token` utility class.

## Using tokens in Tailwind

All custom properties are registered in the `@theme inline` block, which makes them available as Tailwind utility classes via the `--color-*` prefix:

```html
<!-- semantic token as a Tailwind class -->
<section class="bg-[var(--feat-bg)] text-[var(--feat-ttl)]">

<!-- or via the registered color utility -->
<section class="bg-feat-bg text-feat-ttl">
```

```css globals.css
@theme inline {
  --color-feat-bg: var(--feat-bg);
  --color-feat-ttl: var(--feat-ttl);
  /* ... */
}
```

<Warning>
Always reference semantic tokens (`--feat-bg`, `--hero-ttl`, etc.) in component code rather than primitive values like `--background` or hardcoded hex values. This ensures the component responds correctly to both color mode and accent color changes.
</Warning>
